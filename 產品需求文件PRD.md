# **產品需求文件 (PRD) \- 校園二手書循環平台 (vibe)**

| 欄位 | 內容 |
| :---- | :---- |
| **產品名稱** | vibe \- 校園二手書循環平台 (Campus Book Loop) |
| **文件版本** | V1.2 |
| **文件狀態** | 詳細設計版 (基於單一學校架構) |
| **目標對象** | 特定校園師生 (單一學校獨立運作) |
| **核心價值** | 安全信任 (Trust)、資源永續 (Loop)、誠信交易 (Honesty) |

## **1\. 專案背景與願景 (Project Vision)**

### **1.1 產品定位**

本平台為**專屬單一學校**開發之封閉式二手書交易系統。所有使用者皆須經過該校校友/師生身份驗證，確保交易環境的純粹性與安全性。

## **2\. 身份與隱私模組 (Identity & Privacy)**

### **2.1 實名綁定與隱私顯示**

* **實名制後台**：註冊時必須填寫「真實姓名」與「學號/教職員編號」，並通過 @\*.edu.tw 或@shsh.tw信箱驗證。  
* **前端匿名機制**：  
  * 使用者可自定義「暱稱」。  
  * 在商品頁面、評論區與搜尋結果中，僅顯示「暱稱」與「信用等級」。  
  * **交易成立後**：雙方在聊天室中仍可選擇維持暱稱顯示，僅在必要時（如管理員介入）才核對真實身份。

## **3\. 功能需求規格 (Functional Requirements)**

### **3.1 AI 智慧上架審核 (AI Guardrail)**

* **影像自動辨識**：上傳書籍照片時，串接 AI 影像辨識 API (如 Google Vision API)：  
  * **過濾非書籍圖片**：若偵測非書本、講義或文字內容，禁止上架並提示「請上傳清晰的書籍照片」。  
  * **違禁品過濾**：自動識別色情、暴力或校規禁止之物品影像。  
* **書況輔助**：AI 自動提取封面文字（書名、作者、ISBN）自動填寫表單資訊。

### **3.2 交易機制：時間協商流程 (Rescheduling)**

為解決「非惡意爽約」，設計以下正式協商流程：

1. **發起修改**：任一方可在面交前 2 小時透過系統點擊「請求修改時間」。  
2. **對方確認**：  
   * **接受**：系統更新面交時間，T-24h 提醒邏輯根據新時間重新計算。  
   * **拒絕**：維持原時間或進入「取消交易」流程。  
3. **次數限制**：單筆交易僅限修改 2 次，超過則建議直接取消。

### **3.3 交易憑證：LINE 閉環驗證 (LINE Verification Loop)**

本專案利用 LINE Official Account 作為核心互動媒介：

* **買方憑證 (Proof of Item)**：  
  * 面交現場拿到書後，買方需拍攝「書籍實體照片」並傳送至系統 LINE 帳號。  
  * 系統透過 LINE Webhook 接收照片並自動標註該筆交易「買方已領取」。  
* **賣方憑證 (Proof of Payment)**：  
  * 賣方確認收到金錢（現金或匯款）後，於 LINE 點擊「我已收款」按鈕。  
* **自動結案**：  
  * 當系統同時收到「買方照片」+「賣方確認」，自動將商品轉為 已成交 並跳出評價邀請。

## **4\. 管理端需求 (Admin Control)**

### **4.1 管理後台功能**

* **一鍵下架 (Quick Takedown)**：管理員在瀏覽商品或查看檢舉時，可直接點擊「下架」並輸入理由（如：內容不符、違禁品）。系統將自動發送 LINE 通知給賣家並扣除信用分。  
* **爭議仲裁單**：查看買方上傳的照片與雙方對話，判定爽約責任。  
* **身份管理**：可根據學號將特定使用者永久黑名單。

## **5\. 流程圖 (Workflow)**

### **5.1 完整交易路徑**

1. **上架**：賣家拍照 \-\> **AI 審核通過** \-\> 商品上傳成功。  
2. **媒合**：買家發起請求 \-\> 雙方透過系統聊天室約定時間。  
3. **協商 (Optional)**：一方臨時有事 \-\> 發起時間修改 \-\> 對方同意。  
4. **T-24h 確認**：雙方於 LINE 收到通知並點擊「會準時赴約」。  
5. **現場成交 (核心環節)**：  
   * 買家拿書 \-\> 拍照傳到 LINE。  
   * 賣家收錢 \-\> 於 LINE 點擊「已收款」。  
6. **結案**：系統自動更新信用分數、雙方互評。

## **6\. 技術規格補充 (Technical Specifications)**

* **LINE Messaging API**：  
  * 使用 Rich Menus 提供快速操作入口（如：我的商品、修改時間、上傳憑證）。  
  * 利用 Quick Replies 加速交易確認流程。  
* **資料保護**：  
  * 所有真實姓名與學號資料必須加密儲存。  
  * 聊天室紀錄僅保留 90 天，過期自動清除以保護個人隱私。

## **7\. 成功指標 (Success Metrics)**

* **AI 精準度**：AI 誤判率（將書誤認為非書）需低於 5%。  
* **憑證完成率**：買賣雙方皆能順利在 LINE 完成「照片上傳」與「收款確認」的比率。  
* **爭議率**：引入協商機制後，進入人工仲裁的交易比例。
